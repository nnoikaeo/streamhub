# Version Management

Semantic versioning and release process.

## Version Format

`MAJOR.MINOR.PATCH` (e.g., 1.2.3)

- **MAJOR:** Breaking changes
- **MINOR:** New features
- **PATCH:** Bug fixes

## Tagging

```bash
git tag v1.0.0
git push origin v1.0.0
```

## Release Notes

Create in GitHub releases with:
- What's new
- Bug fixes
- Breaking changes
- Migration guide
